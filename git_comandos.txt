Para crear una nueva rama
git branch nombre_de_rama

para cambiar a otra rama
git checkout nombre_de_rama

si cambio de rama se va borrar todo lo que hice en esa rama pero al regresar a la rama anterior se recupera lo que había hecho en esa rama

si quiero unir a la rama principal otra rama primero regresar a la rama principal
git checkout master
después
git merge nombre_rama

lo ideal después de unir una rama con otra es borrar la rama para hacerlo es
git branch -d nombre_rama
cuando no se a unido a ninguna rama debes de confirmar que si deseas borrarla o sino usar
git branch -d nombre_rama -f
el -f lo fuerza

para crear una rama y moverse a la rama en un solo comando
git checkout -b nombre_rama

Merge o uniones de ramas con conflictos sucede cuando en dos ramas trabajo el mismo archivo
después de resolver el conflicto solo hace un commit -am

git stash sirve para guardar algo que se han hecho cambios pero si aún no quieres subir a producción el proyecto con cambios que andas realizando en ese momento, aunque lo ideal es usar ramas
git stash

para ver referencias de stash
git stash list

para recuperar el stash usar
git stash pop

borrar todos stash
git stash clear

cuando se tiene n varios stash se puede recuperar uno en especial con
git stash apply nombre_stash(stash@{2})

borrar ultimo stash
git stash drop
borrar un stash en espesifico
git stash drop nombre_stash(stash@{1}) 

saber un poco que se hizo en el stash
git stash show nombre_stash

ponerle un nombr eo mensaje al stash para saber que tiene
git stash save "nombre_mensaje"

ver más información de los stash
git stash list --stat

git checkout nombre_rama(master)

git branch -d nombre_rama(secundaria)
esto casi no se usa es mejor usar el merge de ramas y resolver conflictos

Cuando se hacen modificaciones a varios archivos y quieres revertir cambios de un archivo en especifico sin usar git checkout -- . que revierte todos
git checkout -- NOMBRE_ARCHIVO.EXTENCION

Si se hace un commit de varios archivos pero quiero separarlos en commits independientes de cada archivo puedo usar lo siguiente
git rebase -i HEAD~NUMERO_DE_COMMITS
después se abrirá información de la cantidad de commits seleccionado y los comandos que podemos usar
ver la imagen para referencia
con la letra "A" puedo escribir en los resultados después cambia el "pick" por "edit" o "e" va unir ese con el anterior commit
salir con
ESC + :wq!
Después hacer:
git reset HEAD^
y después añadir uno por uno
git add NOMBRE_ARCHIVO
y hacer su commit
git commit -m "NOMBRE COMMIT"
Después lo restante en caso de que sean solo 2 pero si se quieren hacer más hacerlo el add de cada uno
git commit -am "NOMBRE COMMIT"
FINALMENTE para terminar el rebase interactivo usar:
git rebase --continue

Salir en caso de que no finalizamos el rebase
